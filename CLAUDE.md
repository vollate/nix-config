This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

This is a modular NixOS Flake configuration based on ryan4yin/nix-config. It uses a custom library to compose NixOS systems from reusable modules, with Home Manager integrated for user-level configuration.

## Development Commands

### Building and Deploying

Use `just` for all common operations:

```bash
# Deploy configuration to current host (tb-amd-6800h)
just deploy

# Deploy to specific host
just deploy hostname

# Debug deployment with trace
just debug

# Update flake inputs and deploy
just update

# Build without deploying
just build

# Check for errors before deploying
just check
```

### Development and Testing

```bash
# Enter development shell (includes nil, nixfmt-classic)
nix develop
# or
just dev

# Format all Nix files
nix fmt
# or
just fmt

# Test configuration in VM
just test-vm

# Show diff between current and new configuration
just diff
```

### Maintenance

```bash
# Clean old generations and garbage collect
just clean

# Show system generations
just generations

# Rollback to previous generation
just rollback
```

### Package Management

```bash
# Search for packages
just search query

# Show package info
just info-pkg package-name
```

## Architecture

### Configuration Flow

The configuration is assembled through a hierarchy:

1. **flake.nix** → Imports custom library (`lib/default.nix`)
2. **lib/default.nix** → Provides `mkHost` helper that composes:
   - Standard modules (`modules/base`, `modules/desktop`, `modules/development`)
   - Host-specific config (`hosts/${hostname}/`)
   - User config (`users/${username}.nix`)
   - Home Manager integration with user home config (`home/${username}/`)
3. **Modules** → Self-contained functionality grouped by category
4. **Host config** → Hardware-specific settings and overrides
5. **Home Manager** → User-level program configurations

### Key Library Functions

Located in `lib/default.nix`:

- `mkSystem`: Low-level function to create NixOS configurations with custom modules
- `mkHost`: High-level wrapper that includes standard modules (base, desktop, development) automatically

When defining a new host in `flake.nix`, use `myLib.mkHost` with these parameters:
- `hostname`: System hostname
- `username`: Primary user
- `desktop`: Desktop environment (e.g., "plasma", or null for servers)
- `system`: Architecture (default: "x86_64-linux")
- `overlays`: Package overlays to apply
- `specialArgs`: Additional arguments to pass to modules

### Module System

Modules are organized by responsibility:

- **modules/base/**: Core system configuration (boot, networking, users, locale, Nix settings, security)
- **modules/desktop/**: Desktop environment (audio/PipeWire, fonts, X11, Plasma)
- **modules/development/**: Development tools (programming languages, Docker, virtualization)
- **modules/editors/**: Editor configurations (if present)

Each module directory contains:
- `default.nix`: Entry point that imports sub-modules
- Individual `.nix` files for specific functionality

### Host-Specific Configuration

Each host in `hosts/` has:
- `default.nix`: Main host configuration, imports other host files
- `hardware-configuration.nix`: Auto-generated by nixos-generate-config (hardware scan)
- `hardware.nix`: Manual hardware optimizations (e.g., graphics vendor)
- `networking.nix`: Host network configuration
- `home.nix`: Host-specific home manager configuration (if needed)

Host configs can override module defaults and add host-specific settings.

### Home Manager Integration

User configurations in `home/${username}/`:
- Integrated via Home Manager NixOS module (not standalone)
- Automatically receives `inputs`, `hostname`, `username`, `desktop` as arguments
- Organized into subdirectories: `programs/`, `shell/`, `desktop/`

### Overlays

Overlays are defined in `overlays/` and allow modifying or adding packages. The `overlays/default.nix` exports a list of overlays that are applied via the `overlays` parameter in `mkHost`.

Example: `neovim-with-nodejs.nix` provides a Neovim package with Node.js support.

### Private Configuration

The `private/` directory contains a local-only flake input for secrets and sensitive configuration. This is imported but not committed to git (via submodule or gitignore). The `secrets` from `private/secrets.nix` are passed as `specialArgs` to all modules.

## Special Considerations

### Chinese Mirror Configuration

This configuration uses Chinese mirror substituters for faster downloads in China. Binary cache substituters are configured both in:
- `flake.nix` nixConfig (affects flake operations)
- Host `nix.settings.substituters` (affects system operations)

### Graphics Vendor

The custom option `nixosVollate.graphicsVendor` (in host config) determines graphics driver configuration. Currently supports "amd".

### nix-ld for FHS Compatibility

Enabled system-wide via `programs.nix-ld` for running non-NixOS binaries (e.g., VSCode Server, dynamically linked executables). Uses `nix-ld-rs` package.

### SSH Access

OpenSSH is configured for remote access (VSCode Remote, etc.) with:
- Root login disabled
- X11 forwarding enabled
- Longer connection timeouts for remote development
- Port 22 open in firewall

## Adding New Functionality

### Adding a New Host

1. Create directory in `hosts/new-hostname/`
2. Add hardware configuration: `sudo nixos-generate-config --show-hardware-config > hosts/new-hostname/hardware-configuration.nix`
3. Create `hosts/new-hostname/default.nix` (use existing host as template)
4. Add to `flake.nix`:
   ```nix
   nixosConfigurations.new-hostname = myLib.mkHost {
     hostname = "new-hostname";
     username = "vollate";
     desktop = "plasma"; # or null for server
     overlays = overlays;
     specialArgs = { inherit secrets; };
   };
   ```

### Adding a New Module

1. Create module file in appropriate category (base/desktop/development)
2. Import in the category's `default.nix`
3. If creating a new category, add to `mkHost` function in `lib/default.nix`

### Adding Programs to Home Manager

1. Create `.nix` file in `home/${username}/programs/`
2. Import in `home/${username}/programs/default.nix`
3. Configure using Home Manager options (see Home Manager manual)
